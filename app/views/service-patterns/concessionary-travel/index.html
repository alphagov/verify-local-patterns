This page is for people who work in local authorities. It outlines a common pattern used for delivering older person's concessionary travel services effectively. Read more about service patterns.

This pattern is specific to England. Concessionary travel passes work differently in Scotland, Wales and Northern Ireland.

This is made possible through the English National Concessionary Travel Scheme. The Department of Transport has published guidance for councils about this scheme.

What is the user need?

As someone who is eligible to travel for free
I need to apply for a concessionary travel permit
So that I am able to travel for free

What do councils need from users to deliver the service?

  • proof that the user lives in the area
  • proof the user is old enough to be entitled to a pass
  • a passport style photograph of the user to put on the pass

What is the user journey?

At a high level, these are the steps users should go through.

  1. Awareness

    Users need to find out about their entitlement to concessionary travel. They may find out in a variety of ways, for example:

    • signage on buses and bus stops
    • signage in a local library
    • from local, district, borough and county councils
    • through word of mouth
    • a search on the internet

    Local authorities should try make sure the right information is being provided through these channels.

  2. Check you are the right age

    Residents in England can get a bus pass for free travel when they reach the female State Pension age, whether they’re a man or a woman.

    Residents of London can travel free on buses, tubes and other transport when they’re 60 or older, but only within London.

    This means that users may need to check whether or not they are old enough for concessionary travel. There's a page for this on GOV.UK. Councils should link to this page on their service's start page. (See step 3.)

  3. Find the right council

    Once the user has found out they can get a free travel pass, they need to get to the right council's website to apply. Users might start by typing 'older person bus pass' or a related term into a search engine, or they might start on their local council's website.

    The older person's bus pass page on GOV.UK should send them to the right council's website once they enter their postcode. Councils should make sure the postcodes that they serve are linked to the right page on their website and email any corrections to local-direct-admin@digital.cabinet-office.gov.uk.

    Users may also go straight to the website for the council that they think they need to get a travel pass from. This makes it very important that councils clearly state which area their concessionary travel service serves as the user may be in the wrong place. Read the next section for details on this.

  4. Service start page

    Now the user is in the right place, the page on the council's website should communicate:

    • what the pass will allow its holder to do
    • when the user is allowed to travel using the pass (The national bus concession in England is available at any time on a Saturday, Sunday or bank holiday, and from 9.30am to 11pm on any other day although some authorities offer extended travel times.)
    • where the pass is valid
    • how the pass will be used
    • how long the process of applying will take
    • when they should expect to receive their bus pass once they've applied
    • that they'll need to provide a passport style photograph
    • that they'll need to login or signup using GOV.UK Verify
    • what to do if they need assistance with the application
    • what to do if they can not complete the application
    • links to renewals and replacements services
    See the example service start page

  6. Confirm your address and eligibility

    GOV.UK Verify will send the council the user's address, as known by the user's chosen identity provider. On returning from the GOV.UK Verify process, the council should ask the user to confirm whether or not this is their current address. This page can also ask the user to confirm they want the council to send the bus pass to the address they've received through GOV.UK Verify.

    See the example page asking users to confirm their name and address

    If the user says the address returned is not correct, they are likely to be a new resident whose address has not been updated with their identity provider. The user should then be asked to enter their current address and warned that they must not enter false information as this is a civic offence. If the travel pass is physical, councils will get extra confidence in the address by mailing the pass to the address given. Councils should mention they will be using the address for this purpose on the confirmation page.

    See the example page asking users to enter their current address

    Once the user has confirmed their address, the council should check that the given address is in the area the council serves. If the address is not in the area, the council should tell them they are not eligible in this area.

    See the page telling the user they are not eligible because of their address

    If their address is eligible, the councils should check their date of birth. This task does not need to be visible to the user it can be performed as a background function

    If the user is not eligible based on their date of birth councils should tell them they are not eligbile yet and communicate when the user will be eligible.

    See the page telling the user they are not eligible because of their age

    If the user is not eligible because they already have a pass, the council should tell them and also link to the 'Renew or replace your bus pass' service. The page should also include a link to contact the council if the user believes there has been a mistake.

    See the page telling the user they are not eligible because they already have a pass

  7. Upload passport style photograph

    Bus passes need to have a photograph of the person who owns them, so that they can't be used by anybody else. It's a bus driver's responsibility to check the photo matches the user.

    Users can upload a digital passport style photograph for use on the bus pass, but they may need to be instructed how. The council website should ask the user whether or not they need instructions, and then step them through the process if they do.

    Step through the example guide for how to take and upload a passport style photograph

  8. Sending the bus pass to the user

    Once the user has uploaded their photograph, the council should tell the user their application is complete and reiterate when they can expect to receive their bus pass and that they will be posting it to the address they previously confirmed.

    The accompanying letter should also provide information on how, when and where the bus pass can be used, and what to do in case of a lost or stolen pass.

    See the example page telling the user when they can expect to receive their pass

  9. Contacting users

    You may want to capture users contact details so they can be contacted if things go wrong with their application, or for renewal reminders.

    You should allow users to select their prefered contact method and only contact them in this way in future.

    See the example page asking users to provide their prefered contact details.

  10. Use bus pass

    Bus passes are a physical plastic card that users can use on any bus in England. They either have to show it to the driver or scan on a reader when they get onto the bus.

    In London, the pass is an Oyster card which the user scans on a reader.

  11. Renewals

    To reduce the possibility of fraud, bus passes have expiry dates which must be no more than 5 years from the date of issue. (Specification 1.31 from the Department for Transport guidance.)

    Councils should send a notification to the user's preferred channel when their bus pass is about to expire.

    In the future, councils will be able to send this reminder using GOV.UK Notify.

    See the example notification letter to remind users to renew their pass (page in progress)
