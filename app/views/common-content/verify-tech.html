The interactions between GOV.UK and the council's website are as follows

  • Council service creates authentication request SAML and records authentication request ID in its session store
  • User goes through Verify process
  • GOV.UK Verify matching service receives matching request
  • GOV.UK Verify matching service responds 'no match'
  • (Here if we were matching, the service would interact with a data store, but for new applications this is not needed.)
  • GOV.UK Verify matching service receives `unknown user account creation` and responds 'success'
  • Council service back end receives assertion with name and address
  • Council service's SAML session store checks request ID to make sure it's the right request
  • Council service's front end receives user details
